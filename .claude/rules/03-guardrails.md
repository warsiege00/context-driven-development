# O que não fazer

- Não sugira trocar libs ou padrões que já têm ADR registrado sem antes
  apontar o ADR existente e perguntar se a decisão deve ser revisada
- Não crie abstrações antecipadas — implemente o mínimo que a spec exige
- Não atualize docs de discovery ou architecture automaticamente —
  esses são mantidos manualmente
